# Rule Engine: документация по условиям доступа

## Общая структура правила

Правило (`rule`) состоит из списка условий (`conditions`) и эффекта
(`effect`):

``` json
{
  "conditions": [
    {
      "type": "ip",
      "operator": "in",
      "value": "192.168.1.0/24"
    }
  ],
  "effect": true
}
```

## Логика выполнения

Все условия объединены через **AND**:

-  Если хотя бы одно условие не выполняется → доступ запрещён\
-  Если все условия выполняются → применяется `effect`

```
effect = true  → ALLOW  
effect = false → DENY
```    

## Формат условия

``` json
{
  "type": "<тип>",
  "operator": "<оператор>",
  "value": "<значение>"
}
```

  |Поле       |Описание
  |---------- |----------------------------
  |type       |Тип проверяемого параметра
  |operator   |Оператор сравнения
  |value      |Значение для сравнения

## Поддерживаемые типы и операторы

### Пользователь

#### gid

``` json
{
  "type": "gid",
  "operator": "equals",
  "value": 1001
}
```

  |Оператор     |Описание
  |------------ |----------
  |equals       |равен
  |not_equals   |не равен

#### groups

``` json
{
  "type": "groups",
  "operator": "contains",
  "value": "sudo"
}
```

  |Оператор   |Описание
  |---------- |-------------------------------
  |equals     |наличие группы
  |contains   |пользователь состоит в группе

### Сеть

#### ip

``` json
{
  "type": "ip",
  "operator": "in",
  "value": "192.168.1.0/24"
}
```

  |Оператор   |Описание
  |---------- |-------------------
  |equals     |точное совпадение
  |in         |входит в CIDR
  |not_in     |не входит в CIDR

### Хост

#### hostname

``` json
{
  "type": "hostname",
  "operator": "regex",
  "value": "^prod-.*"
}
```

  |Оператор     |Описание
  |------------ |--------------
  |equals       |равен
  |not_equals   |не равен
  |regex        |по регулярке

### Время

#### timestamp

``` json
{
  "type": "timestamp",
  "operator": "between",
  "value": "09:00-18:00"
}
```

  |Оператор   |Описание
  |---------- |------------------
  |between    |диапазон времени

### День недели

#### weekday

``` json
{
  "type": "weekday",
  "operator": "in",
  "value": ["mon", "tue", "wed"]
}
```

  |Оператор   |Описание
  |---------- |-----------------
  |equals     |конкретный день
  |in         |список дней

## Ограничения

1.  Все условия объединяются через AND\
2.  Тип value должен соответствовать type\
3.  Не все операторы применимы ко всем типам
